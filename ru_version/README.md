# Шифратор Argon2id и AES-GCM

Надежный, автономный и прозрачный инструмент шифрования, разработанный для пользователей, требующих абсолютного контроля над своими данными. Написан на Python с использованием `tkinter` и библиотеки `cryptography`.

## Философия

Эта программа была создана для "разумных параноиков". Она не доверяет никому: ни облаку, ни операционной системе, ни даже разработчику.
*   **Никаких бэкдоров:** Здесь нет мастер-паролей или ключей восстановления.
*   **Никаких облаков:** Всё работает локально на вашем компьютере.
*   **Никакой жалости:** Если вы потеряете пароль или конфигурацию KDF, ваши данные будут математически невосстановимы.

## Техническая безопасность

Мы используем общепринятые, проверенные криптографические примитивы. Никакой "самопальной" криптографии.

1.  **AES-256-GCM (Режим Галуа/Счетчика):**
    *   **Конфиденциальность:** 256-битное шифрование.
    *   **Целостность (Аутентифицированное шифрование):** Режим GCM вычисляет криптографический тег (контрольную сумму) для данных. Если файл поврежден, изменен вирусом или взломан хакером, расшифровка завершится ошибкой `InvalidTag`. Вы никогда не получите поврежденные или подмененные данные.
    *   **AAD (Дополнительные аутентифицированные данные):** Позволяет привязать зашифрованные данные к определенному контексту.

2.  **Argon2id (Функция деривации ключа):**
    *   Победитель конкурса хеширования паролей (Password Hashing Competition).
    *   Защищает от атак методом перебора (brute-force) на GPU/ASIC, требуя значительного объема оперативной памяти и процессорного времени для получения ключа шифрования из вашего пароля.

## Возможности

### 1. Шифрование одиночного файла
Загружает весь файл в оперативную память. Лучше всего подходит для файлов малого и среднего размера, когда требуется максимальная скорость.
*   *Примечание:* Ограничено доступным объемом вашей оперативной памяти.

### 2. Потоковое шифрование (поблочное)
Читает и записывает файлы небольшими фрагментами (например, по 64 КБ).
*   **Низкое потребление памяти:** Может шифровать файлы любого размера (даже терабайты) на машине с 2 ГБ ОЗУ.
*   **Маскировка размера:** Зашифрованные файлы дополняются (padding) для выравнивания по размеру блока, что слегка скрывает точный размер файла.

### 3. Манифест (Шифрование папок)
Рекурсивно шифрует всю структуру каталога.
*   Файлы переименовываются в случайные строки (например, `a8z9_k2.enc`) и перемещаются в единую папку "Vault" (Хранилище).
*   Создается файл `manifest.json` для сопоставления случайных имен с исходной структурой папок.
*   *Совет профи:* Зашифруйте `manifest.json` отдельно для максимальной безопасности.

### 4. Стеганография
Скрывает зашифрованные архивы или файлы внутри обычных изображений (JPG, PNG).
*   Результат выглядит как обычное изображение.
*   Позволяет извлечь как чистое изображение, так и секретные данные.

### 5. Шредер файлов
Безвозвратно удаляет файлы или папки.
*   Перезаписывает данные случайным шумом перед удалением.
*   Переименовывает файлы перед удалением, чтобы скрыть их названия от инструментов восстановления файловой системы.
*   Настраиваемое количество проходов (от 1 до 7).

### 6. Утилиты
*   **Генератор паролей:** Создает надежные пароли с высокой энтропией.
*   **Проверка целостности:** Вычисляет хеши SHA-256 для проверки целостности файлов.
*   **Экспорт сессии:** Сохранение текущей конфигурации, пароля и хеша в файл JSON (используйте с осторожностью!).

## Конфигурация (Вкладка Настройки)

Вкладка **Settings** (Настройки) имеет решающее значение. Она управляет параметрами KDF (Функции деривации ключа).
*   **Memory Cost (Затраты памяти):** Сколько оперативной памяти требуется для разблокировки файла. Больше = надежнее защита от брутфорса, но медленнее открытие.
*   **Iterations (Итерации):** Сколько раз повторяются математические вычисления.
*   **Chunk Size (Размер фрагмента):** Управляет использованием памяти в режиме потокового шифрования.

**⚠️ ВАЖНО:** Чтобы расшифровать файл, вы должны использовать **точно те же** настройки KDF (Итерации, Память, Длина ключа), которые использовались при шифровании. Если вы измените их, вы не сможете расшифровать свои старые файлы даже с правильным паролем.
